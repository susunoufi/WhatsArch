שלב 0 — נקודת ההתחלה

המשתמש נכנס לאתר WhatsArch (באינטרנט, על Railway). הוא רואה את מסך הניהול עם אופציית העלאה לענן.

במחשב, האתר מנסה לגלות ברקע אם יש Agent מותקן:

// בכל טעינת דף
fetch('http://localhost:11470/status')
  .then(r => r.json())
  .then(d => agentAvailable = true)
  .catch(() => agentAvailable = false)

אם אין Agent — מוצגת אופציה להתקנה.
בטלפון — רק ענן (Agent לא רלוונטי).

ארכיטקטורה: הממשק תמיד באתר. ה-Agent רץ על המחשב של המשתמש ברקע — כמו Spotify או Discord.

שלב 1 — Toggle מקומי / ענן

כשה-Agent מזוהה, מופיע Toggle בראש מסך הניהול. המשתמש בוחר את מצב העבודה.

מקומי: הכל רץ על המחשב. קבצים לא עוזבים את הדיסק. בלי הגבלת גודל.

ענן: מעלים ZIP לשרת. עיבוד ב-Railway. API חיצוני לתמלול ו-AI.

// Agent מזוהה?
if (agentAvailable) {
  showModeToggle(); // מציג toggle
} else {
  showCloudOnly(); // רק ענן
}
בטלפון: ה-Toggle לא מופיע. תמיד ענן. אין דרך להריץ Agent על טלפון.

שלב 2 — לחיצה על "התקן Agent"

אם ה-Agent לא מותקן, המשתמש לוחץ על כפתור ההתקנה. יורד קובץ EXE קטן.

ה-EXE מריץ installer שמתקין:

1. Python (אם אין)
2. ffmpeg (חילוץ אודיו/וידאו)
3. Ollama (אופציונלי — AI מקומי)
4. Dependencies (faster-whisper, Flask...)
5. מוריד מודל Whisper (~500MB)

// ה-installer הוא Flask app זמני
// נפתח בדפדפן של המשתמש
// localhost:11471 (פורט זמני)
// מראה UI יפה עם סטטוס לכל שלב
אחרי התקנה: ה-Agent נרשם כשירות Windows ועולה אוטומטית עם הפעלת המחשב. המשתמש לא צריך לעשות שום דבר.

שלב 3 — Agent רץ ברקע

ה-Agent עולה ב-localhost:11470 כשירות רקע שקט. המשתמש לא רואה כלום — בדיוק כמו Spotify שרץ ברקע.

האתר מזהה את ה-Agent ומציג את ה-Toggle.

# Windows Task Scheduler
# מריץ את agent.py בכל התחברות
schtasks /create /tn "WhatsArch Agent"
  /tr "pythonw agent.py"
  /sc onlogon /rl highest

ה-Agent חושף API endpoints שהאתר קורא להם ישירות מהדפדפן (CORS מוגדר).

שקיפות: האתר שולח בקשות ל-localhost:11470 כשצריך עיבוד מקומי, ול-Railway כשצריך ענן. המשתמש לא מודע לזה.

שלב 4 — בחירת תיקיית צ'אט

המשתמש לוחץ "פתח תיקיית צ'אט" באתר. נפתח חלון בחירת תיקייה של Windows (native dialog).

הוא בוחר את התיקייה שייצא מוואטסאפ.

// האתר שולח בקשה ל-Agent:
POST localhost:11470/browse/folder

// Agent פותח native file picker:
tkinter.filedialog.askdirectory()

// מחזיר:
{ "path": "C:\\Users\\me\\WhatsApp" }

// Agent יוצר symlink (לא מעתיק!):
POST localhost:11470/upload/local
{ "path": "C:\\Users\\me\\WhatsApp" }
Symlink: במקום להעתיק GB של מדיה, ה-Agent יוצר קיצור דרך לתיקייה המקורית. אפס כפילויות.

שלב 5 — עיבוד מקומי

הצ'אט מופיע בניהול עם תג "מקומי". המשתמש לוחץ על כפתורי העיבוד.

כל העיבוד רץ על המחשב דרך ה-Agent:

תמלול: faster-whisper (מקומי, חינם)
תמונות: Ollama vision או API (לבחירה)
אינדוקס: SQLite FTS5 מקומי
Embeddings: multilingual-e5-large מקומי

// האתר מזהה שזה צ'אט מקומי:
if (agentChats.has(chatName)) {
  base = 'http://localhost:11470';
} else {
  base = ''; // Railway
}

// כל הקריאות הולכות ל-Agent:
POST ${base}/api/process/start
GET ${base}/api/process/status
ניתוב חכם: האתר יודע אילו צ'אטים מקומיים ואילו בענן, ומנתב אוטומטית לשרת הנכון.

שלב 6 — חיפוש ושימוש

אחרי העיבוד — הצ'אט מוכן. אפשר לחפש, לשאול AI, לצפות בגלריה, אנליטיקס — הכל עובד.

כל הנתונים נשארים על המחשב. שום דבר לא נשלח לענן.

// חיפוש בצ'אט מקומי:
GET localhost:11470/api/search?
  chat=MyChat&q=hello

// AI chat מקומי (Ollama):
POST localhost:11470/api/ai/chat/stream
  { chat: "MyChat", question: "..." }
פרטיות מלאה: במצב מקומי + Ollama, אפס נתונים יוצאים מהמחשב. חיפוש, AI, הכל local.
